### 基本設定

|參數|意思|例子|
|----|----|----|
|--model|模型名稱|facebook/opt-6.7b|
|--path|權重存放路徑，沒有會自動從 HuggingFace 下載|./weights|
|--offload-dir|disk offload 的資料夾|./offload|
|--prompt-len|輸入的 prompt 長度（tokens）|512|
|--gen-len|要生成幾個 token|128|
|--cut-gen-len|快速 debug 用，提早停止生成|5|

### 效能關鍵參數 ⭐

|參數|跟你的研究的關係|
|----|----|
|--attn-sparsity|設定 attention sparsity，例如 0.9 代表只用 top 10% 的 attention，跟你的 hot/cold 想法直接相關|
|--compress-cache|開啟 KV cache 壓縮（4-bit），可以當你的一個實驗對照組|
|--cpu-cache-compute|把 attention 計算移到 CPU 做，減少 GPU↔CPU 搬移次數|
|--overlap|讓資料搬移和計算 overlap，隱藏搬移延遲，跟你的 prefetch 想法相關|

### Debug 用

|參數|意思|
|----|----|
|--debug-mode fewer_batch|只跑少量 batch，快速確認能動|
|--debug-mode breakdown|輸出各階段時間細項，看哪裡慢|
|--verbose|輸出更多|
|log--log-file|把結果存到檔案|

### 實驗可能會用到的指令

```bash
# Baseline：KV cache 全在 GPU
python3 -m flexllmgen.flex_opt \
    --model facebook/opt-6.7b \
    --percent 100 0 100 0 100 0 \
    --prompt-len 512 --gen-len 128 \
    --debug-mode breakdown
```

## 最常用的執行指令

```bash
# 跑小模型 OPT-1.3B，會自動從 Hugging Face 下載
python3 -m flexllmgen.flex_opt --model facebook/opt-1.3b
```

```bash
# 跑 OPT-30B，使用 CPU offload
python3 -m flexllmgen.flex_opt \
  --model facebook/opt-30b \
  --percent 0 100 100 0 100 0
```

```bash
# 如果記憶體不夠，壓縮 weight
python3 -m flexllmgen.flex_opt \
  --model facebook/opt-30b \
  --percent 0 100 100 0 100 0 \
  --compress-weight
```

```bash
# 全部 weight offload 到 disk，適合 CPU/GPU RAM 不夠時
python3 -m flexllmgen.flex_opt \
  --model facebook/opt-30b \
  --percent 0 0 100 0 100 0 \
  --offload-dir ~/flexllmgen_offload_dir
```

--percent 六個數字的意思是:
weight GPU%, weight CPU%, cache GPU%, cache CPU%, activation GPU%, activation CPU%  
例如:

```bash
--percent 0 100 100 0 100 0
```

代表 weight 放 CPU，attention cache 和 activation 主要放 GPU。
